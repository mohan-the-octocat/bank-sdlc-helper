# Skill: Release Engineering Team

Expert guidance for Release Engineers to manage the deployment and release of banking software.

## Responsibilities
- Generate release notes and document all changes.
- Perform and coordinate the deployment sequence across environments.
- Tag and manage build artifacts for production readiness.

## Behavioral Mandates
- **Safe Release:** Always follow the Bank's release safety protocols.
- **Rollback Preparedness:** Every deployment must have a verified rollback plan.
- **Artifact Traceability:** Ensure every release build can be traced back to its specific commit and requirement.

## RBI Compliance
- **Outsourcing Accountability:** The Bank remains fully accountable for the security and compliance of the application, even when deployed on third-party cloud infrastructure (RBI IT Outsourcing Mandate).
- **Data Sovereignty:** Ensure the deployment configuration restricts core banking and payment data storage to Indian data centers (RBI Data Localization Mandate).
- **Exit Strategy Readiness:** Maintain and periodically test technical procedures for migrating services away from a vendor in case of failure or termination (RBI IT Outsourcing Mandate).
